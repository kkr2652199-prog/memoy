## 🏗️ **1. 파일 구조 관련 용어**

### **📁 폴더 (Folder/Directory)**

프로젝트의 파일들을 분류해서 보관하는 곳

```
우리 프로젝트 폴더:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 templates/          → HTML 파일 보관
📁 static/            → CSS, JS 파일 보관
📁 core/              → 핵심 기능 Python 코드 
📁 routes/            → 페이지 연결 Python 코드
📁 database/          → 데이터베이스 파일
📁 docs/              → 문서, 설명서
📁 backup/            → 백업 파일
📁 logs/              → 로그 파일
```

| 파일 확장자 | 이름 | 용도 | 예시 |

## 💬 **3. AI에게 요청하는 방법**                                              

### **✅ 좋은 요청 예시**

#### **케이스 1: 새 기능 추가**
```
❌ "파일 드래그 기능 추가해주세요"

✅ "파일 드래그 앤 드롭 기능을 만들어주세요:
   1. static/js/file_drag_handler.js 파일로 로직 생성
   2. static/css/file_drag.css 파일로 스타일 생성
   3. conversion.html에 스크립트 링크만 추가"
```

#### **케이스 2: UI 수정**
```
❌ "탭 디자인 바꿔주세요"

✅ "업종별 탭 디자인을 수정해주세요:
   1. static/css/industry_tabs.css 파일 생성
   2. 기존 conversion.css에서 탭 스타일 제거
   3. conversion.html에 새 CSS 링크 추가"
```

#### **케이스 3: 버그 수정**
```
❌ "버튼이 안 눌려요"

✅ "변환시작 버튼이 클릭되지 않는 문제를 수정해주세요:
   - 위치: conversion.html 또는 conversion.js
   - 증상: 클릭 시 반응 없음"
```

| 용어 | 설명 | 언제 사용 |
|------|------|-----------|

| **분리** | 큰 파일을 여러 작은 파일로 나누기 | "이 기능을 별도 파일로 분리해주세요" |
| **컴포넌트화** | 재사용 가능한 조각으로 만들기 | "탭을 컴포넌트화해주세요" |
| **모듈화** | 기능별로 독립된 파일로 만들기 | "검증 로직을 모듈화해주세요" |
| **리팩토링** | 기능은 그대로, 구조만 개선 | "conversion.html을 리팩토링해주세요" |

## 🎯 **5. 실전 요청 템플릿**

### **템플릿 1: 새 UI 추가**
```
"[UI 이름]을(를) 만들어주세요:
1. templates/components/[이름].html 파일 생성
2. static/css/[이름].css 파일 생성
3. conversion.html에 include 추가

예: "배송정보 입력 폼을 만들어주세요:
    1. templates/components/delivery_form.html 생성
    2. static/css/delivery_form.css 생성
    3. conversion.html에 include 추가"
```

### **템플릿 2: 새 기능 추가**
```
"[기능 이름] 기능을 만들어주세요:
1. static/js/[이름].js 파일 생성
2. conversion.html에 스크립트 추가

예: "엑셀 파일 검증 기능을 만들어주세요:
    1. static/js/excel_validator.js 생성
    2. conversion.html에 스크립트 추가"
```

### **템플릿 3: 스타일 수정**
```
"[대상]의 디자인을 수정해주세요:
1. static/css/[이름].css 파일에서 수정
   (또는 새 파일 생성)
2. conversion.html은 건드리지 마세요

예: "달력의 디자인을 수정해주세요:
    1. static/css/calendar_custom.css 생성
    2. 기존 conversion.css에서 달력 스타일 제거
    3. conversion.html에 새 CSS 링크 추가"
```

### **템플릿 4: 코드 정리**
```
"[파일 이름]이(가) 너무 커요. 리팩토링해주세요:
1. [큰 부분]을 [새 파일 이름]으로 분리
2. [다른 부분]을 [또 다른 파일]로 분리
3. 원본 파일은 최소화

예: "routes/conversion.py가 너무 커요. 리팩토링해주세요:
    1. API 핸들러를 routes/conversion_api.py로 분리
    2. 검증 로직을 core/conversion_validator.py로 분리
    3. 원본은 라우팅만 남기기"
```
