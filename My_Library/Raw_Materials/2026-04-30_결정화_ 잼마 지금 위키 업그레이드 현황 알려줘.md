## 자료별 상세 내용

### 📰 LLM 위키의 개념적 진화 (AI 사서 답변)

'LLM 위키' 시스템은 AI가 단순한 정보 요약을 넘어 고품질의 전문적인 결과물을 생성하기 위해 구축되는 **개인 지식 도서관**을 의미합니다. 이 시스템의 발전 과정은 다음과 같습니다.

*   **과거 AI의 한계:** 과거 AI는 '사모대출이란 무엇인가'와 같은 질문에 위키백과 요약 수준의 답변만을 제공할 수 있었습니다.
*   **LLM 위키의 발전:** 현재 LLM 위키를 통해 AI는 **감정 곡선과 팩트 체크까지 포함된 전문적인 대본**을 생성할 수 있게 되었습니다. 이는 AI 자체가 똑똑해진 것이 아니라, AI 뒤에 '개인 도서관', 즉 'LLM 위키'가 생김으로써 가능해진 변화입니다.
*   **핵심 기능:** 이 시스템은 AI가 자료를 **수집하고 정리하며, 질문에 답변하고, 모순을 점검하는 'AI 사서' 역할**을 수행하도록 설계되었습니다.
*   **지식의 심화:** 도서관에 자료가 축적될수록 AI의 결과물은 더욱 깊어지며, AI가 스스로 유지보수를 담당하여 관리 부담이 줄어드는 것이 목표입니다.

### 🧠 '지식 위키'의 구조 분석 (안드레아스 카파의 LLM 위키 개념 분석)

자료들은 이 지식 기반 구축에 있어 **'지식 관리(Knowledge Management)'**와 **'개인 지식 기반(Personal Knowledge Base)'** 구축을 강조합니다.

*   **정의 변화:** 전통적인 '위키피디아'가 정보의 **'집합체'**였다면, LLM 기반의 '지식 위키'는 **'지식의 연결 구조와 의미론적 관계'**를 이해하는 **'지식 엔진'**에 가깝습니다.
*   **핵심 목표:** 단순히 정보를 저장하는 것을 넘어, 정보 간의 관계를 이해하고(Semantic Understanding), 새로운 추론을 가능하게 하는 데 중점을 둡니다.

### 🛠️ LLM 위키 구축 방법 (상위 0.1프로만 쓰는 현 시점 Ai를 가장 똑똑하게 쓰는 방법)

LLM 위키를 구축하기 위한 준비물과 과정은 다음과 같습니다.

*   **준비물:**
    1.  AI 코딩 도구 (Claude Code 또는 Codex)
    2.  Karpathy의 LLM 위키 개념서
    3.  마크다운 파일 뷰어인 Obsidian
*   **구축 과정:**
    1.  프로젝트 폴더 생성 및 AI 코딩 도구를 연다.
    2.  Karpathy 개념서를 붙여 넣고 지침서를 요청하여 AI가 폴더 구조(소스, 위키)와 지침서 파일을 자동으로 생성한다. (약 2분 소요)
    3.  **자료 입력:** AI에게 '수집'이라고 말하고 URL을 붙여 넣으면, AI가 원본을 소스 폴더에 저장하고 핵심 내용을 추출하여 위키 폴더에 정리된 페이지를 생성하며, 기존 페이지들과 자동으로 연결된다.
