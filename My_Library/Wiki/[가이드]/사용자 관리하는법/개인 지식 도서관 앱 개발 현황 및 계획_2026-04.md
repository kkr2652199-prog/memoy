## 핵심 요약
My_Library 프로젝트는 FastAPI, SQLite, Ollama(llama3), D3.js를 기반으로 구축된 개인 지식 도서관 애플리케이션이다. 2026년 4월 19일 기준으로 172개의 자료가 데이터베이스에 저장되어 있으며, `nomic-embed-text-v2-moe` 모델을 사용하여 모든 자료의 임베딩이 완료되었다. BM25+벡터 하이브리드 검색, 임베딩 생성, LLM 제공자 선택 등 핵심 기능들이 구현되었으며, 최근 대규모 리팩토링, UI 개선, 용어 통일, YouTube 자막 fallback 기능 강화, 알림 시스템 개선, 그리고 벡터 검색 1~3단계 구현을 완료했다. 현재는 신규 자료 섭취 시 자동 임베딩 기능을 개발 중이며, 향후 하이브리드 검색 테스트, YouTube 자막 기능 안정화, UI 설정 업그레이드 등 다양한 기능 개선을 계획하고 있다.

## 상세 분석
**프로젝트 개요:**
*   **프로젝트명:** My_Library (D:\MONEY lol\My_Library)
*   **백업일:** 2026-04-19
*   **기술 스택:** FastAPI, SQLite, Ollama (llama3), D3.js (지식그래프)
*   **데이터베이스:** `data/library.db` (자료 172개, 임베딩 172/172 완료)
*   **임베딩 모델:** `nomic-embed-text-v2-moe` (Ollama, 768차원, 512토큰 제한)
*   **LLM:** OpenAI, Claude, Gemini, Local(llama3) 순서의 fallback 시스템 (Gemma4는 설치되었으나 미사용)

**핵심 파일 및 기능:**
*   `app/core/search.py`: BM25+벡터 하이브리드 검색 구현 완료 (1139행)
*   `app/core/embedding_engine.py`: 임베딩 생성 및 저장 로직 (`_build_text` 400자 제한)
*   `app/llm/embedding_client.py`: Ollama `/api/embed` 클라이언트
*   `app/db/database.py`: `material_embeddings` 테이블 포함
*   `app/core/ingest.py`: 자료 섭취 로직, `ingest_material` 함수에서 commit 발생
*   `app/llm/provider.py`: LLM 제공자 선택 로직
*   `app/core/health_check.py`: 중복 알림 버그 수정 완료 (is_read 조건 제거)
*   `app/api/routes_settings.py`: 알림 전체 읽음/삭제 API 추가
*   `app/core/url_fetchers.py`: YouTube 자막 fallback (youtube-transcript-api → yt-dlp)

**완료된 주요 작업:**

*   용어 통일 Phase 1~2b (UI, 프롬프트, 위키, 템플릿)
*   그래프 범례 수정
