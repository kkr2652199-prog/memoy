# 리팩토링

## 설명

리팩토링은 소프트웨어의 외부 동작을 변경하지 않으면서 내부 구조를 개선하여 가독성을 높이고 유지보수성을 향상시키는 체계적인 과정이다.

유형/분류:
- 코드 리팩토링: 개별 함수, 클래스, 모듈의 가독성과 효율성을 높이는 작업
- 아키텍처 리팩토링: 시스템 전체의 설계나 구성 요소를 재조정하여 구조적 문제를 해결하는 작업
- 데이터 리팩토링: 데이터베이스 스키마나 데이터 구조를 개선하여 효율성과 일관성을 높이는 작업

핵심 원리: 소프트웨어의 외부 동작을 변경하지 않는 불변성을 유지하며, 작은 단위로 점진적인 개선을 통해 위험을 최소화하는 것이다.
