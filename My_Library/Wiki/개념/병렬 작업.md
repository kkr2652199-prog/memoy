# 병렬 작업

## 설명

병렬 작업은 여러 작업을 동시에 실행하여 전체 처리 시간을 단축하고 효율성을 높이는 컴퓨팅 방식이다.

유형/분류:
- 에이전트 동시 관리: 여러 AI 에이전트 작업을 동시에 시작하고 모니터링하여 전체 프로젝트 진행을 가속화한다.
- 충돌 방지 병렬 실행: 여러 작업이 동시에 실행될 때 발생할 수 있는 데이터 접근 충돌이나 상태 불일치를 워크트리 같은 메커니즘으로 방지한다.
- 클라우드 기반 병렬 실행: 로컬 자원 제약 없이 클라우드 서버에서 장시간 또는 고성능이 요구되는 작업을 병렬로 수행하여 지속성과 확장성을 확보한다.

핵심 원리: 병렬 작업은 단일 프로세서가 아닌 여러 처리 장치나 스레드를 활용하여 작업을 분할하고 동시에 처리함으로써 전체 작업 완료 시간을 단축한다. 이때, 작업 간의 의존성을 관리하고 자원 충돌을 방지하는 메커니즘이 필수적이다.
