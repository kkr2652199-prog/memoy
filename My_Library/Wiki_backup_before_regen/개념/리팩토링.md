# 리팩토링

## 설명

리팩토링은 소프트웨어의 외부 동작을 변경하지 않으면서 내부 구조를 개선하는 과정입니다. 이는 코드의 가독성, 유지보수성, 확장성을 높여 소프트웨어의 품질을 향상시키는 것을 목표로 합니다. 개발자는 리팩토링을 통해 기술 부채를 줄이고, 버그 발생 가능성을 낮추며, 새로운 기능을 더 쉽게 추가할 수 있도록 코드를 정리합니다. 주로 작은 단위의 변경을 반복적으로 수행하며, 이는 소프트웨어 개발 수명 주기 전반에 걸쳐 지속적으로 이루어집니다. [[코드 개선]] [[소프트웨어 공학]] [[유지보수]]

제목: 개인 지식 도서관 앱 개발 현황 및 계획
요약: My_Library 프로젝트는 FastAPI, SQLite, Ollama(llama3), D3.js를 기반으로 구축된 개인 지식 도서관 애플리케이션입니다. 2026년 4월 19일 기준으로 172개의 자료가 데이터베이스에 저장되어 있으며, `nomic-embed-text-v2-moe` 모델을 사용하여 모든 자료의 임베딩이 완료되었습니다. BM25+벡터 하이브리드 검색 기능을 포함합니다.
