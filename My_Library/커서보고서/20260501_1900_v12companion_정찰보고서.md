## 2. predict_combo.py 알고리즘 분석

- **입력:** `miss_draws` (미당첨 회차 dict 리스트).
- **패턴:** `_analyze_combo_patterns` — 회차별 당첨 6개 번호에서  
  - 모든 **쌍(pair)** 출현 빈도 → `pair_scores` (정규화)  
  - **연번 개수** 평균 `consec_target`  
  - **합(sum)** 평균 `sum_target`
- **생성:** `random.sample(1..45, 6)` 후보에 대해  
  - `abs(sum(cand)-sum_target) <= 25`  
  - `tier1_filter(cand)` 통과  
  - 점수 = 후보 6개 번호 쌍들의 `pair_scores` 합  
- **출력:** `army3_combo_predict` → `brain_tag`/`method`는 역전 조합 문자열(후 `_retag`로 `v11조합두뇌`).

**요약:** 통계적 쌍·합·연번 휴리스틱 + **의사난수** 기반 샘플링. **외부 LLM API 호출 없음.**

## 5. run / offset 과 차별화 (corr, Rule 14)

- **v12_run:** 연속 구간·런 길이 등 **시계열 인접** 축.
- **v12_offset:** 회차 간 **간격·자기상관** 축.
- **companion (후보):** 번호 간 **공출/균형/구간** 축 — run·offset과 **같은 피처를 직접 공유하지 않음**.

**corr:** 본 정찰에서 **수치 상관계수는 미계산**(대량 `nums` 벡터 정렬·정규화 필요). 설계상 **상관 < 0.5 기대**는 타당하나, **패치 전 `numpy` corrcoef 경고가 백테 로그에 존재** → companion 도입 시 **수치 검증 스크립트(임시 후 삭제)** 로 별도 검증 권장.
