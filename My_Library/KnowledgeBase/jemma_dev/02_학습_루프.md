# 2. 학습 루프 (한 사이클)

모든 코딩·지식 작업에 아래 **5단계**를 **짧게** 적고 시작한다. (채팅 상단에 붙여도 됨)

1. **읽기 (Read)**  
   - 변경하려는 **파일·함수**를 **최소 범위**로 읽는다.  
   - `.cursor/rules`의 `my-library`, `change-scope`, `collaboration-terms`를 **한 번** 확인한다.

2. **가설 (Hypothesis)**  
   - “이 수정이 **한 가지** 문제만 해결하는가?”  
   - “`content` 원본·`wiki_body` 덮어쓰기 **금지**에 걸리지 않는가?”

3. **최소 변경 (Minimal change)**  
   - 한 PR/한 대화 = **최대 2~3파일** 이내를 기본으로 한다. (넓게 훑는 리팩터는 **에스컬레이션**)

4. **검증 (Verify)**  
   - Python 백엔드 변경 시:  
     `python -c "from app.main import app; print('app OK')"`  
   - DB 스키마 건드렸다면: 사용자에게 **부사수 턴**에서 `init_db`/PRAGMA 확인.

5. **기록 (Log)**  
   - `KnowledgeBase`에 **1~3줄**만: 날짜, 무엇을 배웠는지, 다음에 할 일. (긴 일지는 사용자 선택)

**반복:** 검증이 실패하면 **가설을 갱신**하고, 같은 파일을 **추측으로** 여러 번 고치지 않는다.
