# 4. 부사수(Cursor) 에스컬레이션 — 언제 넘길까

다음 **어느 하나**라도 해당되면, 잼마는 **코드를 대량으로 수정하지 말고** 사용자에게 말한다:  
> “이건 **Cursor(부사수) 턴**에서 `grep`/파일 읽기 후 패치하는 게 안전하다.”

- `library.db` / SQLAlchemy 모델 **스키마 변경** (`ALTER`, 새 컬럼)
- `init_db`·마이그레이션·`PRAGMA`가 필요한 작업
- `app/static`·`index.html`·캐시버스트·여러 JS 파일에 걸친 UI
- `health_check`, 스케줄러, 인제스트 파이프 **동작 경로**를 바꾸는 작업
- “전역으로 일괄” 치환 (위키·DB·룰 **금지**에 해당할 수 있음)
- **한 번** 시도했는데 **같은 오류가 반복**될 때 (추측 금지 원칙)

**넘길 때 준비할 정보 (사용자가 Cursor에 붙이기 쉽게):**

- 목표 한 문장
- **파일 경로** (추정 말고, 읽은 뒤의 **정확한 path**)
- **기대한 동작** vs **실제 동작**
- 이미 실행한 **검증 명령**과 **출력**

**반대로 잼마가 **유지**해도 좋은 것:**  
- `jemma_dev/`·`10_Wiki/개념`에 있는 **문서** 초안, 학습 루프 **체크리스트** 작성  
- “어디를 봐야 할지” **질문** 정리, 요구사항 **구조화** (→ 부사수가 구현에 집중)
