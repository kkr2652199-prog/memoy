# LM Studio 전제 + 이름 4층 (고정, 혼동 금지)

## 로컬 LLM (우리 팀 기준)

- **로컬에서 쓰는 쪽 = LM Studio만** 사용한다. **Ollama는 이 팀 루틴에 끼우지 않는다** (다른 PC/문서에 나와도 “우리 표준”과 혼동하지 말 것).
- LM Studio는 **API/채팅 UI**로 돌릴 **모델 서버(및 클라이언트)** 역할 — **“잼마”를 그 위에서 쓰는** 그림이 맞다.

## 생각해도 되는 4층 (한 그림)

| 층 | 이름 | 뜻 |
|----|------|-----|
| 1 | **잼마** | LM Studio 쪽 흐름(로컬 모델 + 그 UI/채팅). **앱 런타임 밖** 조력. |
| 2 | **커서 (내부 / 부사수)** | **Cursor IDE 안**의 이 AI — 레포·터미널·패치. |
| 3 | **로하** | **My_Library 웹앱** 안의 채팅/지식 AI — **서버·DB**가 부름. |
| 4 (참고) | **“웹앱”** | 브라우저에서 쓰는 **우리 앱** 전체 (로하가 붙는 곳). |

- **잼마 ≠ 커서 내부 ≠ 로하** — **세션·메모리·API가 다르다.**
- **Ollama ≠ “우리가 쓰는 방식”** — 문서/남이 `ollama run` 얘기해도, **팀 내부 룰은 LM Studio**로 맞출 것.

## 터미널에서 “잼마(= LM Studio 서버)” 호출 — **가능**

잼마 **UI**를 띄우는 건 아니지만, **같은 LM Studio**가 켜 둔 **OpenAI 호환 API**로 터미널에서 요청하면, **잼마가 쓰는 모델과 동일 엔드포인트**에 말을 걸 수 있다.  
**로하**도 앱 설정에서 **같은 `lmstudio_endpoint` + 모델**을 쓰면, **물리적으로는 같은 LM Studio 프로세스**에 붙는 그림이 된다(대화 기록·세션 ID는 **앱 vs 터미널 vs 잼마 UI**마다 다름).

**우리 앱 기본값(참고):** `data/config.json`의 `lmstudio_endpoint` (예: `http://localhost:1234`), `lmstudio_model` (예: `google/gemma-4-e4b`). **LM Studio에서 Local Server 켜기**가 선행.

**PowerShell 예 (한 턴):**

```powershell
$body = @{
  model    = "google/gemma-4-e4b"   # config.json 의 lmstudio_model 과 맞출 것
  messages = @(@{ role = "user"; content = "한 문장으로: 너는 팀의 잼마다." })
  stream   = $false
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Uri "http://localhost:1234/v1/chat/completions" `
  -Method Post -ContentType "application/json; charset=utf-8" -Body $body
```

> **잼마(LM Studio) · 커서(IDE 안) · 로하(웹앱) — 셋은 뇌를 공유하지 않는다. 로컬 모델 경로는 LM Studio로 통일.**
