# 상태 비저장

## 설명

상태 비저장은(는) 시스템이 이전 요청이나 상호작용에 대한 정보를 저장하지 않고, 각 요청을 독립적으로 처리하는 아키텍처 또는 설계 원칙이다.

유형/분류:
- 웹 서비스: RESTful API와 같이 클라이언트의 상태를 서버에 저장하지 않고 각 요청에 필요한 모든 정보를 포함하는 방식.
- 대규모 언어 모델 (LLM): [[Gemma 4]]와 같은 모델이 기본적으로 이전 대화를 기억하지 않고, 매번 새로운 입력에 대해 응답을 생성하는 방식. (단, 대화 맥락은 입력 프롬프트에 포함하여 시뮬레이션 가능)
- 네트워크 프로토콜: HTTP와 같이 각 요청이 독립적으로 처리되며, 서버가 클라이언트의 이전 상태를 유지하지 않는 통신 방식.
- 함수형 프로그래밍: 순수 함수가 외부 상태를 변경하거나 외부 상태에 의존하지 않고, 동일한 입력에 대해 항상 동일한 출력을 반환하는 특성.

핵심 원리: 각 요청은 자체적으로 완전하며, 서버는 요청을 처리하는 데 필요한 모든 정보를 요청 자체에서 얻는다. 이로 인해 시스템의 확장성과 안정성이 향상된다.

관련 도구: [[RESTful API]], [[HTTP]], [[Gemma 4]], [[람다 함수]]
