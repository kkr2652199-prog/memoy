# 컴포넌트화

## 설명

컴포넌트화은(는) 시스템을 독립적이고 재사용 가능한 작은 단위인 컴포넌트로 분해하여 설계하고 구축하는 원칙이다.

유형/분류:
- 소프트웨어 컴포넌트화: 코드의 재사용성과 유지보수성을 높이기 위해 기능을 모듈화된 단위로 분리하는 접근 방식
- UI 컴포넌트화: 사용자 인터페이스를 버튼, 입력 필드 등 재사용 가능한 시각적 요소로 나누어 개발 효율성을 증대
- 비즈니스 컴포넌트화: 비즈니스 로직을 독립적인 서비스 단위로 분리하여 유연한 시스템을 구축하는 방법

핵심 원리: 각 컴포넌트는 고유한 기능을 캡슐화하고 명확한 인터페이스를 통해 다른 컴포넌트와 상호작용하며, 이를 통해 시스템의 유연성과 확장성을 확보한다.
