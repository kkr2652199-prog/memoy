2. 의학/논리추론 — 심인성 쇼크와 승압제
상황 이해
급성 심근경색(심장마비)이 발생하면 심장의 펌프 기능이 급격히 떨어집니다. 혈액을 전신에 충분히 보내지 못하면 **심인성 쇼크(cardiogenic shock)**가 옵니다. 증상은 혈압 급락, 의식 저하, 장기 부전입니다.

왜 노르에피네프린인가?
승압제(혈압을 올리는 약)는 여러 가지가 있습니다.

**노르에피네프린(Norepinephrine)**은 알파-1 수용체를 강하게 자극해서 혈관을 수축시키고 혈압을 올립니다. 동시에 베타-1 효과로 심장 수축력도 약간 높여줍니다.

**도파민(Dopamine)**은 예전에 1차 선택이었지만, 대규모 연구(SOAP II 임상시험, 2010)에서 심인성 쇼크 환자에게 도파민을 쓰면 부정맥 발생률이 2배 높아진다는 결과가 나왔습니다. 이후 가이드라인이 바뀌었습니다.

**에피네프린(Epinephrine)**은 심박수를 과도하게 올려 이미 손상된 심장에 부담을 줄 수 있어 1차로 쓰지 않습니다.

배울 점: 근거기반의학(EBM)
의학에서는 "원래 그렇게 해왔으니까"가 아니라, 대규모 임상시험 데이터에 근거해서 치료법을 바꿉니다. 이것이 근거기반의학의 핵심 철학입니다.

3. 창의력 — 양자역학 중첩을 소네트로
양자 중첩이란?
일상에서 동전은 앞면 또는 뒷면입니다. 하지만 양자 세계에서 전자 같은 입자는 관측하기 전까지 여러 상태가 동시에 공존합니다. 이것이 **중첩(superposition)**입니다.

슈뢰딩거의 고양이 사고실험이 이를 비유합니다. 상자를 열기 전까지 고양이는 살아있으면서 동시에 죽어있는 상태입니다.

소네트 구조
소네트(sonnet)는 14행으로 이루어진 서양 정형시입니다.

셰익스피어식 소네트의 경우 구조는 4-4-4-2 (3개의 4행 연 + 마지막 2행 대구)이고, 각운(rhyme) 패턴은 ABAB CDCD EFEF GG 입니다.

이 문제가 어려운 이유
AI든 사람이든, 과학적 정확성 + 문학적 아름다움 + 형식적 제약을 동시에 만족해야 합니다. 이것은 단순 지식이 아니라 여러 영역의 지식을 통합하는 능력을 시험합니다.

배울 점
창의력이란 무에서 유를 만드는 것이 아니라, 서로 다른 영역의 지식을 연결하는 능력입니다. 양자역학을 아는 것, 소네트 형식을 아는 것, 이 둘을 엮는 것 — 세 번째가 가장 어렵고 가치 있습니다.

4. 코딩 — 시간 복잡도와 Counter
시간 복잡도란?
입력 크기 N이 커질 때, 알고리즘이 얼마나 느려지는지를 나타내는 척도입니다.

O(N)은 선형 — 입력이 2배면 시간도 2배입니다. O(N 
2
 )은 제곱 — 입력이 2배면 시간이 4배입니다. O(logN)은 로그 — 입력이 2배면 시간은 아주 조금만 늘어납니다.

구체적 예시: 빈도수 세기
리스트 [1, 3, 2, 1, 3, 3]에서 각 숫자가 몇 번 나오는지 세야 한다고 합시다.

나쁜 방법 — O(N²)

Copyfor each unique element:       # N번
    count how many times it appears  # N번 순회
# 총 N × N = N² 연산
좋은 방법 — O(N), Counter 사용

Copyfrom collections import Counter
data = [1, 3, 2, 1, 3, 3]
result = Counter(data)
# result = {3: 3, 1: 2, 2: 1}
Counter는 내부적으로 **해시 테이블(딕셔너리)**을 사용합니다. 리스트를 딱 한 번만 순회하면서 각 요소를 해시맵에 기록하므로 O(N)입니다.

해시 테이블이 빠른 이유
해시 함수가 데이터를 고유한 주소로 변환합니다. 마치 도서관에서 책을 찾을 때, 처음부터 끝까지 훑는 게 아니라 분류번호로 바로 찾아가는 것과 같습니다. 평균적으로 삽입과 조회가 모두 O(1), 즉 상수 시간입니다.

배울 점
코딩 면접과 실무에서 가장 중요한 사고방식은 "같은 결과를 더 빠르게 얻을 수 있는 자료구조가 있는가?"입니다. 대부분의 최적화는 적절한 자료구조 선택에서 시작됩니다.

종합 정리
이 네 문제에서 관통하는 하나의 원칙이 있습니다. 바로 **"거대한 문제를 작은 조각으로 분해하라"**는 것입니다. 수학에서는 100을 4와 25로 쪼갰고, 의학에서는 쇼크의 원인을 기전별로 분리했고, 창의력에서는 형식과 내용을 각각 만족시킨 뒤 합쳤고, 코딩에서는 중첩 루프를 해시맵 한 번 순회로 단순화했습니다.
