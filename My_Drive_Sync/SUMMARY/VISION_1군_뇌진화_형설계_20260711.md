## 1. 뇌 구성 (형 정의)

- **명칭:** 1군 7뇌 (실질 **학습·진화 축은 5뇌** + fusion/hyena/lead1 등 — **업데이트·명명 정리 필요**)
- **5뇌 학습 축 (핵심):** stat, markov, llm, lstm, (+ hyena 메타) — 각 뇌가 **독립적으로** 과거→예측→결과→분석→다음 예측

---

## 2. 형이 처음 설계한 루프 (회차별 진화)

```
[회차 N]
  ① 과거 회차(1..N-1) 학습·패턴 축적
  ② N회 미래 예측 (5세트 등)
  ③ 당첨 확정 → 적중/미적중 채점
  ④ 각 뇌별 미적중·적중 **상세 분석**
     - 번호별 연간성·튀는 번호·동시 출현·갑자기 나온 번호 등
  ⑤ 분석 결과를 해당 뇌의 **다음 회차(N+1) 예측**에 반영
  → ⑥ N+1로 반복
```

### 회차별 데이터 성숙도

| 회차 | 형 기대 |
|------|---------|
| **1회** | 과거 없음 → 사실상 랜덤에 가까운 예측 |
| **2~3회** | 분석 페이지 볼 것 거의 없음 (과거 부족) |
| **10·100·1000회** | 번호별 빅데이터 축적 → 패턴 형성 |
| **1100회** | 상당한 과거 기록 → 다음 회차 예측에 **뇌마다 축적된 지식** 반영 |

**형의 「진정한 진화」 정의:**  
1000회 예측했다면, **각 뇌 상세 페이지에 999회차 분량의 과거 data가 계속 적립**되어 있어야 함.

## 3. UI/UX 비전 (형 요구)

- **뇌별 상세 페이지** — 회차별 타임라인
  - 3회차부터: 1·2회 적중/미적중 기록 표시
  - 1231회차: 1~1230 회차 예측·채점·분석 이력 누적
- **분석은 필수** — 「예측 → 꽝 → 끝」이 아님
- **뇌마다 독립 분석** — fusion 명예만이 아니라, stat/markov/llm/lstm 각자의 미적중 세트 분석

## 4. 현재 앱 vs 형 비전 (2026-07-11 fact)

| 항목 | 형 비전 | 현재 앱 (확인됨) |
|------|---------|------------------|
| 회차별 예측 저장 | ✅ | ✅ `lotto_predictions` (brain_tag별) |
| 당첨 후 채점 | ✅ | ✅ `refresh_prediction_scores` |
| **회차별 피드백 분석 저장** | ✅ 매 회차 | ⚠️ `analyze_prediction_feedback` — **백테스트·일괄 백필용**. **실운영 채점 훅에는 미연결** |
| 뇌별 미적중 세트 심층 분석 | ✅ (연간성·동시출현 등) | ⚠️ stat/markov에 **경량 피드백(last_n=20)** 만. **뇌별 전용 분석 페이지 없음** |
| 뇌별 999회 누적 상세 UI | ✅ | ❌ 대시보드·명예의전당·두뇌상태 위주. **회차별 뇌 타임라인 페이지 없음** |
| miss_analysis 뇌 | ✅ (사각지대 학습) | ⛔ `ENABLE_SPECIAL_BRAINS=False` **컷닝** |
| 가중치 진화 | 뇌별 학습+분석 기반 | Hedge **숫자 가중치만** (last_n=50), lstm 누수로 왜곡 |
| LSTM 학습 | 매 회차 walk-forward | **50회마다** ckpt 재사용 → **누수** |

**형 말씀 「예측했어 꽝이네 끝, 분석 없음」→ largely 맞음** (실운영 UX·훅 기준).
