### v12_snake — **포함한다** (또한 **v12_hyena 출력**도 합의에 사용)

`v12_snake_predict_sets`는 **hyena까지 `extend`된 `fresh`**를 인자로 받는다 (`v12_engine.py` 311행). 합의 루프는 **리스트 내 모든 `brain_tag`**에 `get_v12_brain_weights()` 스칼라를 곱한다.

```110:118:d:\MONEY lol\My_Library\app\lotto3\v12_snake.py
    consensus: dict[int, float] = {n: 0.0 for n in range(1, 46)}
    for p in v12_predictions:
        tag = str(p.get("brain_tag", ""))
        w = float(snake_weights.get(tag, 1.0))
```

→ **하위 4뇌·lstm·fusion·hyena**가 **모두** snake 풀(Top 15) 점수에 기여.

### 제거 시 판단

- **hyena**: 하위 4를 빼면 **20/30≈67%** 세트가 사라지고, 합의 점수에서 **stat/run/offset/contrarian 가중치 분량이 통째로 소거** → **15숫자 풀·5005 랭킹이 변함** → 단독 성능 수치와는 별개로 **메타 출력은 변할 수밖에 없음**.  
- **snake**: 동일 + **hyena 5세트**는 유지되는 설계라면, 하위만 제거 시 **합의 입력 스펙트럼**이 줄어 **점수 분포·조합 탐색**이 바뀜.

*(본 보고서는 **실제 제거 AB 실행 없음** — 코드 구조상 **영향 가능성 높음**으로만 기술.)*

## Q4. Hedge 가중치 비율 (패치 G 종료 값 기준·산술)

|----|--------|

| **전체 7뇌 합** | **303.35** |

- **하위 4 / 7뇌 전체** = 30.46 / 303.35 ≈ **0.1004 (약 10.0%)**

**hyena 합의에 실제 참가하는 6뇌**(hyena 호출 시 `fresh` — 하위4+lstm+fusion):

- 하위 4 합 30.46 · lstm+fusion 합 172.89 · **6뇌 합** 203.35  
- **하위 4 / 6뇌(hyena 입력)** = 30.46 / 203.35 ≈ **0.1498 (약 15.0%)**

**해석**: 스칼라만 보면 **합의 점수 총량 중 하위 4의 “가중치 몫”은 ~10%(테이블 기준) 또는 hyena 전용으로 ~15%**. 다만 **번호 하나에 대한 실제 점수**는 “해당 번호를 포함하는 세트 수 × 해당 뇌 가중치”이므로, **번호별 기여도는 시나리오 의존** — 위 비율은 **가중치 자원 배분의 상한 프록시**로 보면 된다.

## Q5. 결론 및 교체 유형 제안

| 항목 | 판단 |
|------|------|
| **상위 4만으로도 DB 평균 적중은 높음** | Q1 avg 전 구간 **~2.17** vs Q2 **~0.79** — **동일 백테에서 ‘단독 행’만 필터한 수치**이므로 **상·하위 역할 분리는 분명**. |
| **하위 제거가 메타에 미치는 영향** | hyena/snake **합의 정의상 하위 4 포함** → 제거 시 **점수 필드·Top15·조합 후보가 변함**. **상위 뇌 단독 SQL 성능과 무관하게**, **파이프라인 출력은 바뀔 수 있음**. |
| **가중치 관점** | 하위 4 스칼라 합이 **전체 ~10%**, hyena 6뇌 입력 기준 **~15%** — **무시할 수준은 아님**. |

**교체·개편 제안 (코드 변경 없이 개념만)**

1. **합의 노이즈 축소**: 평균 적중이 기대(~0.8) 수준인 뇌를 **합의에서 제외**하거나 가중치 하한을 두는 방식은 **별도 패치**에서 검증 필요.  
2. **교체 후보 유형**: **PMF 품질이 검증 가능한 뇌**(상위 정찰에서 내부 학습·융합에 유리했던 유형) — 예: **시계열/앙상블 쪽 보조**, 또는 **1군과 중복되지 않는 신규 휴리스틱**(실제 구현은 미착수).  
3. **실무적 다음 단계**: 하위 제외 **시뮬레이션**(코드 분기 또는 오프라인 재합의) 없이는 **정량 확정 불가** — 본 문서는 **SQL 단독 필터 + 코드 리딩**까지만 확정.
