def _v12_apply_entropy_cluster(
    pmf: dict[int, float],
    entropy_w: dict[int, float],
    cluster_w: dict[int, float],
) -> dict[int, float]:
    """entropy x cluster 가중치 곱하기 (1군 v5 핵심)."""
    weighted: dict[int, float] = {}
    for n in range(1, 46):
        weighted[n] = (
            float(pmf.get(n, 0.0))
            * float(entropy_w.get(n, 1.0))
            * float(cluster_w.get(n, 1.0))
        )
    total = sum(weighted.values())
    if total > 0:
        return {n: v / total for n, v in weighted.items()}
    return {n: 1.0 / 45.0 for n in range(1, 46)}

## Q4. “통일”이 필요한 부분 / 3군 우위

### 3군이 1군 대비 **부족·불일치**

1. **entropy/cluster 미적용 버그(수준 불일치)** — 1군과 동일 수학 경로가 아님.  
2. **입력 뇌 의미**: LLM ↔ **combo** 슬롯 교체 — “통일” 시 **의미 정합**(같은 역할인지) 정의 필요.  
3. **회피**: 3군만 1군 fusion·당첨 이중 회피 → **난이도·세트 수**에 영향(백테 lstm 행 수 차이 등과 연관 가능).

### 3군이 1군 대비 **발전**

- **1군 fusion 세트 차별화** (중복 합의 완화).  
- **직전 당첨 Jaccard 회피** (패치 I, 컨닝 방지).  
- **brain_tag / DB** 3군 전용 분리.

### 통일 방향 권고

- **1차(저위험)**: 3군에서 **`get_entropy_weights(combined_pmf)`**, **`get_cluster_weights(training_draws, pmf_after_entropy)`** 로 **1군과 동일 시그니처**로 수정 → “벡터 퓨전 v5” 정의 일치.  
- **2차(설계)**: LLM vs combo **역할 문서화** 후, 필요 시 1군도 동일 회피 정책 검토(대칭) 또는 3군 회피 임계만 조정.
