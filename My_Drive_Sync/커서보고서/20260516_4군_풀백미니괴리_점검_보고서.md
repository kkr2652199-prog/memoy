## 종합 진단 — 미니 1.50 vs 풀백 0.83 괴리 원인

### 🥇 1순위: **모델 가중치 상태 불일치 (walk-forward vs replay)**

- 풀백: draw N마다 `load → finetune → predict → update_model` (**당시 weights**)
- 미니/live 재실행: 디스크의 **최종 weights**(864회차까지 누적)에서 load 후 draw N용 finetune
- **동일 draw 재호출 시 seq set0이 6적중** (800,820,840) — DB 기록(0~2적중)과 **정반대**
- struct는 y_hat 결정론적 → DB=live **일치**

### 🥈 2순위: **미니 1100~1222 측정 조건 오염**

- 미니 보고서 seq **1.499**는 **651s live predict** (DB walk-forward 아님)
- 장시간 연속 predict + struct `update_models` 디스크 덮어쓰기 + **파일 lock** (1115 ensemble 실패) → **비정상 세션**
- 이번 STEP 3 재현: 1200+ **seq avg 0.70** — **1.50 재현 불가**

### 🥉 3순위: **부분 풀백(5~864) walk-forward 성적은 랜덤 근처**

- seq **0.8279** (old fullback 0.798과 유사, +0.03)
- ensemble **0.7865** (old 0.7905와 유사)
- **에이스 개선 top-k가 walk-forward에서 드라마틱한 uplift 없음**
- p벡터는 sharp하나 **online finetune drift + 확정형 top-12**가 과거 회차 재현성 훼손

### 결론

| 측정 | seq avg | 신뢰도 |
|------|---------|--------|
| 미니 1100~1222 (이전) | 1.499 | ❌ **과대·재현 불가** |
| live 재현 1200+ (이번) | 0.70 | ✅ replay 조건 |
| partial walk-forward 5~864 | 0.828 | ✅ **유일한 정공법** |

**다음 clean run 전 필수:** DB reset + 모델 삭제 + **단일 프로세스** + walk-forward만으로 성적 산출.  
**미니 live predict 단독 수치는 풀백 대용 불가.**
