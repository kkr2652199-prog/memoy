### 3-3. Hyena Commander 핵심 코드

```python
# v12_engine.py — 가중 합의 (하위4는 consensus에서 제외, 패치 H)
def _v12_compute_consensus_score(all_predictions, weights):
    for p in all_predictions:
        tag = p["brain_tag"]
        w = weights.get(tag, 0.0)
        if tag in _EXCLUDE_LOWER4_FROM_CONSENSUS:  # stat/run/offset/contrarian
            w = 0.0
        for n in p["nums"]:
            consensus[n] += w

# Top-15 풀 → 15C6=5005 전수 평가
pool_nums = _v12_select_candidate_pool(consensus, 15)
ranked = _v12_evaluate_combinations(pool_nums, consensus)  # 5005 combos

# tier1_filter + 당첨회피(Jaccard<0.4, 최근3회) → 5세트
return _v12_build_sets_from_ranked(ranked, all_predictions, n_sets, target_draw_no)
```

**합의 방식**: 번호별 **가중 투표(consensus score)** → Top-15 번호 풀 → **15C6 전수 합산 점수** → tier1 + win_avoid 필터 → 1세트 deterministic + 4세트 weighted random(top50)

**필터**: `tier1_filter`(1군 7대), `v12_pass_win_avoid`(최근 당첨 Jaccard<0.4), Dead Zone 후처리

### 3-4. 가중치

- **동적**: `v12_models.update_v12_weights()` — Hedge η=2.0, `new_w = base × exp(η × avg_match/6)`, clip 100
- **DB**: `lotto_brain_weights_army3` (패치 J 로그 테이블 별도)
- **Hyena consensus**: DB 가중치 병합, 하위4 consensus 가중치=0 (패치 H)
- **Fusion**: 4뇌 PMF + DB가중 + entropy×cluster (1군 모듈 재사용)

### 3-5. 3군이 1.502인 비결

| 요인 | 기여 |
|------|------|
| **15C6=5005 전수탐색** | ★★★ 핵심 — 번호 풀에서 최적 6조합을 수학적으로 찾음 |
| **하위4 consensus 제외** | ★★ — avg 0.80 뇌가 Commander를 오염시키지 않음 |
| **상위뇌 PMF fusion** | ★★ — lstm+fusion이 고품질 PMF 제공 |
| **동적 Hedge 가중치** | ★ — lstm/fusion/hyena에 가중치 집중 (최종 hyena=100) |
| **뇌 수 8개** | △ — 하위4 단독은 랜덤 수준, **수가 아니라 hyena 구조** |

**결론**: 특정 1~2뇌(lstm, fusion) + **hyena 5005 Commander**가 성능의 90%. 하위4는 fusion PMF 입력용 보조.
