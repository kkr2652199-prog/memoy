## STEP7 — 위험 분석 (4문항)

### Q1. markov 제거 시 하이에나 / 퓨전 / 캐시 영향?

- **하이에나**: `_v12_compute_consensus_score`에서 `weights.get(tag, 0.0)` 사용 → **태그가 없으면 기여 0**. DB에 남은 `v12_markov` 가중치는 무해하나, **의도적 제거 후에도 DB 행이 남으면** 가중치>0이면(과거 캐시 재사용 시) 이론상 합의에 섞일 수 있음 → **캐시/신규 생성 경로 정리와 짝 필요**.
- **퓨전**: `v12_fusion.py` / `v12_fusion_v5.py`가 **`army3_markov_prob_vector(training_draws)`** 로 **markov PMF를 직접 사용**함 → **기저 `v12_markov` 행만 없애도 퓨전은 동작 가능**. 반대로 **`predict_markov.py` 또는 해당 함수를 제거**하면 퓨전 **깨짐(Y)**.
- **캐시**: 현재 `len(existing) >= 45` 주석상 **기저 7×5**. markov 기저 제거 시 **6×5=30 + hyena5 + snake5 = 40** → **45→40 하향 조정 필요(Y)** (미조정 시 캐시 미스/과다 재계산).

### Q2. DB legacy 처리 권장?

### Q3. v12_offset 한글명을 STEP2-FINAL에 동시 적용 가능? + markov 제거와 충돌?
