## 1) 번호 선택 메커니즘

### 결론: **[가중랜덤]** (과거 데이터 기반 score/weight 계산 후, `random.choices` 등으로 비복원 샘플링)

1군은 **순수 `random.sample(1~45)`만 쓰는 구조가 아님**. 대부분의 뇌가 DB 당첨 이력(`draws`)에서 **번호별 가중치·확률 벡터**를 먼저 계산한 뒤, **가중 비복원 추출**로 6개를 고른다. 다만 **seed 고정 없이** Python `random` 모듈을 쓰는 구간이 많아 **매 호출마다 결과가 달라진다**.

| 뇌 | 점수 계산 | 6번호 선택 | 분류 |
|---|---|---|---|
| stat | 빈도·recency·미출·동반출현 (`predict_statistical.py`) | `random.choices` 가중 비복원 (`187:192`) | 가중랜덤 |
| markov | 전이행렬 + **Random Walk**(난수) (`38:60`, `117`) | 상위25 후보에서 `random.choices` (`149:154`) | 가중랜덤 + 벡터 단계도 난수 |
| llm | LLM API 또는 stat fallback | fallback 시 stat과 동일 (`predict_llm.py` 16-25, 44-48) | LLM: 외부 비결정 / fallback: 가중랜덤 |
| lstm | `get_lstm_prob_vector(draws)` | `random.choices` (`engine.py` 115-125) | 가중랜덤 |
| **fusion(벡터퓨전)** | 4뇌 확률벡터 가중합+엔트로피+클러스터 | ① Top-K Greedy 상위6 **결정론** (`258-273`) ② 나머지 `random.choices` (`283-288`) | **혼합**(1세트 결정론 + 2~5세트 가중랜덤) |
| hyena | 5뇌 25세트 합의점수, 15후보 전수조합 (`97-106`) | ① 1세트: ranked 1위 tier1 통과 **결정론** (`162-166`) ② 2~5세트: top50에서 `random.choices` (`205-207`) | **혼합** |
| lead1(brain7) | 5뇌 25세트 READ-ONLY + reliability 가중 | `random.Random(seed)` seed=`target_draw_no*F1_SEED_MULT` (`560-561`, `378`) | **결정론적 가중랜덤** |

**난수 개입 단계 요약**

- **벡터(기대값) 계산**: stat·lstm·cluster·entropy 구간은 draws+DB 피드백 기반 **결정론**. markov의 `markov_random_walk()`는 매번 `random.choices`/`random.randint` (`57-59`) → **벡터 자체가 비결정론**.
- **6개 조립**: stat/markov/lstm/fusion 2~5세트/hyena 2~5세트 → **seed 없는 `random.choices`**.
- **예외**: fusion·hyena **1세트**, brain7 **전 5세트** → seed 또는 greedy로 **재현 가능**.

## 2) 점수(기대값)의 근거

### 결론: **과거 회차 DB 집계 + (일부) 백테스트 피드백 DB** — 하드코딩 상수만으로 번호를 고르지 않음

### stat (`get_statistical_prob_vector`, `11:87:My_Library/app/lotto/predict_statistical.py`)

- **입력**: `_get_draws_before(target)` — `lotto_draws` 당첨 6번호 (`data_service.py`, 컨닝 방지)
- **계산**:
  - 지수 감쇠 빈도: `exp(-0.02 * gap_idx)` (`23`)
  - 미출 보너스: gap≥50 → ×1.3, gap≥30 → ×1.15 (`38-41`)
  - 최근 5회 hot: 2회+ → ×1.2 (`49-51`)
  - 최근 200회 동반출현 상위30쌍 보너스 (`53-68`)
  - 피드백: `frequent_traps` ×0.8, `frequent_hits` ×1.15 (`71-81`, `feedback.get_feedback_summary`)
- **정규화**: 합=1.0 확률 벡터

### markov (`build_transition_matrix`, `get_markov_prob_vector`, `12:98:My_Library/app/lotto/predict_markov.py`)

- **입력**: 연속 회차 전이 (decay 0.02), Laplace smoothing 0.5 (`24`)
- **Random Walk**: 최근 회차 6번호 각각 시작점, 80스텝 — **난수 전이** (`76`)
- **피드백**: trap ×0.8, hit ×1.15 (`85-90`)

### lstm (`get_lstm_prob_vector`, `engine._lstm_predict_sets`)

- **입력**: draws → LSTM 모델 추론 (실패 시 uniform 1/45)
- **선택**: lstm_vec 가중 `random.choices` (`engine.py` 101-125)

### fusion 벡터퓨전 (`_vector_fusion_predict`, `139:339:My_Library/app/lotto/fusion.py`)

- stat_vec + markov_vec + llm_vec(순수 LLM 세트 역산) + lstm_vec
- **가중치**: `lotto_brain_weights` DB (`_load_brain_weights_from_db`, `feedback.py` 546-571) — 실패 시 `SEED_WEIGHTS` fallback (`feedback.py` 18-21)
- **보정**: Shannon 엔트로피 (`predict_entropy.py` 10-56), K-Means `random_state=42` (`predict_cluster.py` 52)

### hyena (`predict_hyena.py`)

- **입력**: stat~fusion 25세트 (DB 또는 동일 회차 fresh)
- **합의점수**: Σ(뇌가중 × 번호 등장) (`59-77`)
- **후보**: 상위 15번호 → C(15,6)=5005 전수 합산 (`97-106`) — **완전 결정론적 ranking**

### brain7/lead1 (`predict_brain7.py`)

- **입력**: DB의 5뇌 25세트 READ-ONLY + `_brain_number_reliability()` (`559`)
- **가중**: reliability 기반 `generate_f1_v2_strict_sets(flat, rel, seed)` (`560-561`)

## 5) 벡터퓨전두뇌(fusion) 개별 분석

### 다른 뇌 대비 차이

- **단일 패턴이 아닌 메타-뇌**: stat/markov/llm/lstm **확률 벡터**를 합산 (`234-242:My_Library/app/lotto/fusion.py`)
- v4 `_hybrid_predict` (세트 투표, `22-136`)는 레거시; **운영 경로는 v5 `_vector_fusion_predict`** (`engine.py` 336-340)
- 추가 보정: Shannon 엔트로피 + K-Means 클러스터 (`244-248`)

### 융합 구조

```
draws → [stat_vec, markov_vec, llm_vec, lstm_vec]
     → × VECTOR_WEIGHTS (DB lotto_brain_weights)
     → fused_vec
     → entropy + cluster 보정
     → Top-K Greedy 1세트 + 가중 random 4세트 + tier1_filter
```

### 융합 가중치: 고정 vs 동적

- **동적 (DB)**: `_load_brain_weights_from_db()` — Hedge 백테스트 `update_brain_weights` (`run_backtest` `607-608:engine.py`)
- **fallback 고정**: `SEED_WEIGHTS` stat=1.5, markov=1.0, llm=2.5, lstm=2.5 (`feedback.py` 18-21)
- **실패 두뇌 제외**: lstm uniform 또는 llm 전부 fallback 시 해당 키 제거 후 재정규화 (`222-225:fusion.py`)
- fusion 자체는 `lotto_brain_weights`에 **행 없음** (의도적, 주석 `218-219:fusion.py`)

### 테스트 시 관측 (2026-07-18)

- LLM 홀딩 → llm_vec 무효, VECTOR_WEIGHTS에서 llm 제외 (스크립트 stderr 확인)
- 5세트 중 **1세트만** fused_vec 상위6 greedy로 **재현 가능**, 나머지는 가중랜덤

## 최종 한 줄 판정

**1군은 [가중랜덤] 이다.**

1. **순수랜덤 아님**: stat/markov/lstm/fusion/hyena 모두 `lotto_draws` 과거 당첨·전이·동반출현·피드백에서 **score/weight/prob**를 계산한다.
2. **결정론적 기대값(top-k만) 아님**: 6번호 최종 조립에 **seed 없는 `random.choices`**가 다수이며, markov는 **확률 벡터 생성 단계부터 Random Walk 난수**를 쓴다 — 2회 호출 테스트에서 stat/markov/lstm/fusion 전체 세트 **전부 불일치**.
3. **예외**: brain7(lead1)만 `target_draw_no` 기반 seed로 **5세트 완전 재현**; fusion/hyena **1세트**는 greedy/ranked **결정론**.
