## 2. `lotto_predictions_army4` 구조 (4군·1~3군 비교 참고)

**4군 테이블**은 `app/lotto4/models.py`의 `init_lotto4_db()` 정의와 동일하다.

- **PK**: `id` AUTOINCREMENT  
- **핵심**: `target_draw_no`, `brain_tag`, `num1`~`num6`, `matched_count`, `bonus_matched`, `method`, `reasoning`, `confidence`, `created_at`  
- 지시서의 `set_no`·`numbers`(CSV) 컬럼은 **스키마에 없음**. 세트 구분은 **행 5개**(동일 `target_draw_no`+`brain_tag`) + `reasoning`의 `세트k (풀백테)` 로 식별.

## 3. 백테스트 → UI 실시간 반영

### 3.1 스크립트 (`tools/full_backtest_v13.py`)

- 각 **뇌·회차**마다:
  1. 해당 `(target_draw_no, brain_tag)` 기존 `lotto_predictions_army4` 행 `DELETE`
  2. 예측 5세트마다 `lotto_fullbacktest_army4` + `lotto_predictions_army4` **INSERT**
  3. **뇌 단위 `commit()`** — 웹에서 곧바로 `SELECT` 가능
- `V13_BRAIN_METHOD`로 `method` / `reasoning` 정합 유지
- 무효 세트: 플레이스홀더 번호 `(7,14,21,28,35,42)`, `matched_count=-1`, reasoning에 `무효` 표기
- 재시작: 동일 회차에 **fullback 35행 + predictions 35행** 모두 있으면 스킵
- 진행 로그: **`[진행] n/N 완료 (pct%)`** 형식

### 3.2 API

- `GET /api/lotto4/v13/hall-of-fame` — `lotto_predictions_army4` + `lotto_draws` 조인 (기존과 동일)
- `GET /api/lotto4/v13/predictions/draw/{n}/tier-wins` — 동일 테이블, `matched_count>=3`

### 3.3 프론트 (`app/static/js/lotto4.js`)

- 명예의 전당 탭이 활성일 때 **30초마다 `loadHall()`** 자동 호출 (`ensureHallAutoRefresh`).
- 수동 **「다시 불러오기」** 및 **F5**로도 동일 API 재조회 가능.

### 3.4 실행 중 확인 (텍스트)

- 백테 진행 중 `127.0.0.1:6124` 기동 후 `GET /api/lotto4/v13/hall-of-fame` 호출 시 JSON 정상 응답 확인.
- 초기 구간에는 **3개 이상 적중이 거의 없어** 명예의 전당 **상세 테이블이 빈 상태**일 수 있음. 적중이 발생한 회차부터 행이 쌓임.
- **시범 적중 누적**(뇌별 r1~r5 요약)은 적중이 생기면 숫자가 증가함.
