## 1단계 — 메타 가중치(전 뇌 공통) 튜닝

**대상:** `v13_weights_v2.py`의 `update_v13_v2_weights` — 상수 `HEDGE_REFERENCE_MATCH`, `HEDGE_LEARNING_RATE`, `WEIGHT_CLIP_*` (코드 상단 주석 참고).

- [x] **1-1** 현재 식의 의미 요약 — 아래 **「헤지 식 한눈에」** 참고. 상세는 `v13_weights_v2.py` 주석.
- [x] **1-2** “한 번에 하나만” 첫 슬롯: **`HEDGE_LEARNING_RATE`** (현재 `0.05`). 다음 스윕 후보: `0.04`(덜 출렁) / `0.06`(더 민감). **기준선만** 건드리려면 `HEDGE_REFERENCE_MATCH`(현재 `1.5`).
- [ ] **1-3** 첫 수치 변경 후 검증 (아래 주의 참고)
- [ ] **1-4** (선택) 풀 백테스트 재실행 또는 `v13_ensemble` 줄만 전후 비교 — DB 가중치가 동일하면 단독 뇌 지표는 거의 동일할 수 있음
- [ ] **1-5** 변경 적용 후 로그 한 줄

### 헤지 식 한눈에 (1-1)

- 해당 뇌·해당 회차에 채점된 **5세트 평균 적중**을 `avg_mc`라고 함.
- **업데이트:** `new_w = clamp(old_w + HEDGE_LEARNING_RATE * (avg_mc - HEDGE_REFERENCE_MATCH))`
- **`avg_mc > HEDGE_REFERENCE_MATCH`** 이면 델타가 양수 → 가중치 **증가** 경향.
- **기댓값 힌트:** 무작위 한 장의 기대 적중은 대략 `6×(6/45)≈0.8`. 기준을 `1.5`로 두면 무작위보다 훨씬 잘해야 가중치가 오름.
- **v13_ensemble**은 하위 7뇌 출력을 **DB `current_weight`로 투표**하므로, 헤지 결과가 직접 메타 뇌에 반영됨. 다른 7뇌 단독 `predict()`는 가중치와 무관.

### 1-3 주의 (미니·풀 백테스트)

- 스크립트는 뇌별 **순수 예측 로직** 위주로 지표를 내므로, **헤지 상수만 바꿔도 숫자가 안 바뀌는 것이 정상**이다 (특히 ensemble도 같은 시드·같은 DB 가중치면 동일).
- **헤지 효과 검증**은 (가) 실운용 후 `lotto_weight_log_army4` 추적, (나) 회차 시뮬로 가중치 궤적을 재현한 뒤 ensemble만 재측정, 등으로 한다.

## 리더 메모 — 분석에서 나온 것: 문제 vs 인식

| 구분 | 내용 |
|------|------|
| **실제 버그 (수정함)** | `target_draw_no`에 **당첨이 `lotto_draws`에 없을 때** 채점이 안 되는데도 `update_v13_v2_weights`가 돌면, `matched_count >= 0` 행이 없어 AVG가 비고 **0.0으로 간주**되어 **전 뇌 가중치가 한꺼번에 감쇠**했다. → **`lotto_draws`에 해당 회차가 있을 때만 헤지**하도록 패치 (`v13_weights_v2.py`). |
| **설계 인식 (버그 아님)** | `tools/update_weights_v13.py`(배치)와 `update_v13_v2_weights`(회차 헤지)는 **다른 규칙**이다. 둘을 같은 “튜닝 하나”로 보지 말 것. |
| **튜닝 공백 (기능 부족)** | 미니/풀 백테스트는 **뇌 `predict` 단독 지표**가 중심이라, **헤지 상수만 바꿔도 숫자가 그대로인 것이 정상**이다. 헤지·ensemble 연동을 숫자로 보고 싶으면 **별도 시뮬 스크립트**가 다음 패치 후보다. |

### 다음 패치 우선순위 (지휘안)

1. **P0** ✅ — 미채점 회차 헤지 스킵 (위 적용).
2. **P1** — `HEDGE_LEARNING_RATE`를 `0.04` 등 **한 단계만** 조정 + 운영/로그로 관찰 (또는 사용자 승인 후).
3. **P2** ✅ — `tools/simulate_hedge_ensemble_v13.py`: 회차별 헤지 궤적 + `ensemble.predict(..., weights_override=…)`로 `best_avg` 비교 (`--compare-baseline`).
4. **P3** — `update_weights_v13.py` 실행 시 **헤지와 혼용할지** 문서/플래그로 정책 고정 (덮어쓰기 순서 명시).
