## ① 예측 저장 회차 범위·총 건수

```sql
SELECT MIN(target_draw_no) AS min_draw,
       MAX(target_draw_no) AS max_draw,
       COUNT(DISTINCT target_draw_no) AS distinct_draws,
       COUNT(*) AS total_rows
FROM lotto_predictions;
```

| min_draw | max_draw | distinct_draws | total_rows |
|----------|----------|----------------|------------|
| **3** | **1232** | **83** | **1245** |

> 참고: `distinct_draws=83`이므로 min~max 구간 전체가 채워진 것이 아님. 실제 저장 회차는 **3회 1건** + **1120~1148** + **1180~1232** (1149~1179 구간 gap) 등 **불연속 83회차**이다. 회차당 15행(3뇌×5세트) 기준 83×15=1245와 일치.

## ② brain_tag 종류별 건수

```sql
SELECT brain_tag, COUNT(*) AS cnt,
       MIN(target_draw_no) AS from_draw,
       MAX(target_draw_no) AS to_draw
FROM lotto_predictions
GROUP BY brain_tag
ORDER BY cnt DESC;
```

| brain_tag | cnt | from_draw | to_draw |
|-----------|-----|-----------|---------|
| stat | 415 | 3 | 1232 |
| review | 415 | 3 | 1232 |
| markov | 415 | 3 | 1232 |

> **lstm / llm / lead1 / fusion 등 레거시 태그 없음.** 현행 3예측뇌(stat·markov·review)만 존재. 415÷83=5 → 회차당 뇌별 5세트.

## ③ 채점(matched_count) 범위

```sql
SELECT
  SUM(CASE WHEN matched_count IS NULL OR matched_count < 0 THEN 1 ELSE 0 END) AS ungraded,
  SUM(CASE WHEN matched_count >= 0 THEN 1 ELSE 0 END) AS graded,
  MIN(CASE WHEN matched_count >= 0 THEN target_draw_no END) AS graded_from,
  MAX(CASE WHEN matched_count >= 0 THEN target_draw_no END) AS graded_to
FROM lotto_predictions;
```

| ungraded | graded | graded_from | graded_to |
|----------|--------|-------------|-----------|
| **15** | **1230** | **3** | **1231** |

> 미채점 15건은 전부 **1232회**(미래 예측, `matched_count=-1`). 확정 당첨 없는 회차.  
> 채점 완료 1230건 = 82회차×15세트 (3~1231 중 예측이 존재하는 82회차).
