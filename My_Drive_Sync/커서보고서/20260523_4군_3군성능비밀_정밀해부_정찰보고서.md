- **반환 형식**: `list[dict]` — `{nums, brain_tag, confidence, method, reasoning}`

### 1-B. PMF 수집 (hyena 경로 vs fusion 경로)

**Hyena(`_v12_hyena_predict`)는 PMF를 사용하지 않음.** 세트 기반 가중 빈도 합의:

```python
consensus[n] += brain_weight[tag]  # 세트에 n 포함 시
```

| PMF 제공 뇌 | PMF 함수 | hyena 합의 반영 |
|-------------|----------|-----------------|
| v12_stat (CDM) | `army3_cdm_prob_vector` | **0 (패치 H 제외)** |
| v12_run (cooccur) | 없음 (세트만) | **0 (제외)** |
| v12_offset (sumrange) | `army3_sumrange_prob_vector` | **0 (제외)** |
| v12_contrarian | `army3_constraint_prob_vector` | **0 (제외)** |
| v12_lstm | `army3_lstm_prob_vector` | **DB 가중치 (fallback 2.5)** |
| v12_fusion | Fusion V5 내부 4 PMF | **DB 가중치 (fallback 2.0)** |

**패치 H** (`_EXCLUDE_LOWER4_FROM_CONSENSUS`): stat/run/offset/contrarian **합의 가중치 강제 0**.

→ **Hyena Top-15는 사실상 v12_lstm + v12_fusion 10세트만 결정.**

가중치: `get_v12_brain_weights()` — Hedge η=2.0, `exp(η×score_signal)`, clip 100.

### 1-C. Consensus → Top-N

- **Top-N = 15** (고정, `_v12_select_candidate_pool`)
- **기준**: 가중 합의 점수 내림차순 (동점 시 번호 오름차순)
- **4군 차이**: 4군은 **동일 가중(1.0) freq** 210세트; 3군 hyena는 **DB 동적 가중 + 하위4뇌 제외**

### 1-E. 최종 5세트 선정

**3군 hyena:**
1. **세트1**: ranked 전체 순회 → tier1+win_avoid 통과 첫 조합 (실패 시 top5 우회)
2. **세트2~5**: **top50 가중 랜덤** 30회 → tier1+win_avoid (실패 시 top5 우회)
3. **점수**: `sum(consensus[n] for n in combo)` — 구조 보너스 없음
4. **Jaccard**: 세트 간 **명시적 Jaccard 제거 없음** (used set으로 중복만 방지)

**4군 commander:**
1. struct 필터 통과 조합만 scored
2. **Jaccard 0.4 → 0.5 → 0.6** 단계적 완화
3. win_avoid 별도 적용

### entropy × cluster

```python
final_pmf[n] = combined_pmf[n] × entropy_w[n] × cluster_w[n]
```

- entropy: `app.lotto.predict_entropy.get_entropy_weights(training)`
- cluster: `app.lotto.predict_cluster.get_cluster_weights(target_draw_no)` — **1군 DB**

### 출력

- Top-K Greedy 1세트 + 가중 random 4세트

- **난이도**: **어려움**
- **위험**: cluster는 target_draw 기준 1군 DB — walk-forward OK if read-only

### 3. Hedge 동적 가적치 (DB 진화)
