## 1. 커닝 방지 검증 (코드 확인)

공통 원칙: 당첨 이력 조회는 `_utils.load_draws_before(db_path, N)` → SQL `WHERE draw_no < ?` 로 **N 미만만** 로드한다.

```12:24:d:\3kweon\app\lotto4\brains\_utils.py
def load_draws_before(db_path: str, draw_no: int) -> list[dict[str, Any]]:
    """draw_no **이전** 회차 당첨번호만, 오름차순. 반환: [{draw_no, nums: [6개]}]."""
    ...
            WHERE draw_no < ?
```

### 1.1 seq_brain

| 항목 | 확인 |
|------|------|
| `initial_train(target_draw=N)` | `load_draws_before(db_path, target_draw)` → **draw &lt; N** |
| `predict(draw_no=N)` | `load_draws_before(db_path, draw_no)` → **draw &lt; N**. `len(draws) < WINDOW+1` 이면 `_predict_cpu_random_filtered`(미래 미참조) |
| 최근 50회 입력 | 데이터가 충분할 때 `draws[-WINDOW:]` 만 사용 → **N-50 … N-1** 구간 |
| `update_model(draw_no=N)` | `load_draws_before(db_path, draw_no + 1)` → **N회차까지 포함해 N+1 예측용** |

### 1.2 struct_brain

| 항목 | 확인 |
|------|------|
| `initial_train(target_draw=N)` | `load_draws_before(db_path, target_draw)` → **&lt; N** |
| `predict(N)` 내부 | `update_models(N)`, `_infer_y_hat(N)` 모두 `_history_cut(N)=N`(상한 1223) → `load_draws_before(..., N)` → **&lt; N** |
| MA20 미만 | `_build_features` 에서 `len(s) < 20` 이면 전체 `s` 로 `ma20` 대체 |
| `update_models(draw_no)` (백테 루프 끝) | `full_backtest_v13` 에서 `update_models(target + 1)` 호출 → **target 회차 당첨 반영 후 다음 회차 예측용** (커닝 아님) |

### 1.3 evolution_brain

| 항목 | 확인 |
|------|------|
| `get_dynamic_weights(draw_no=N)` | `for delta in 1..LOOKBACK: tgt = N - delta` → 항상 **&lt; N** 회차만 이력 조회 |
| `update_trust(completed_draw_no=N)` | 완료 회차 N 기준 로그·채점 반영; 다음 가중치는 `get_dynamic_weights(N+1, ...)` 가 **N+1 예측**에 사용 |

### 1.4 gap_brain / diversity_brain / ev_brain

- **gap:** `_history_cut(draw_no)` → `min(draw_no, 1223)` 후 `load_draws_before` → **target 미만** 당첨만.
- **diversity:** `lotto_predictions_army4` 조회 시 `target_draw_no < ?` → **미래 예측 미포함**.
- **ev / ensemble 배치 EV:** `cut = min(draw_no, 1223)`, `load_draws_before(db_path, cut)` → **draw &lt; cut**.

### 1.5 ensemble.py (Commander v2)

서브모듈 `seq_brain.predict`, `struct_brain.predict`, `gap`·`diversity`·`ev` 리스코어 경로가 위와 동일한 컷을 따름. `_batch_ev_scores` 도 동일.  
`predict_struct_vector(draw, db, skip_update=True)` 는 **같은 `draw`에 대해 직전 `struct_brain.predict`가 이미 `update_models`를 수행했다는 전제**로 중복 학습만 생략하며, 미래 당첨을 추가로 보지 않는다.

### 1.6 full_backtest_v13.py 메인 루프 순서

1. 각 `target` 에 대해 **먼저** 7뇌 `predict(target)` — 이때 모든 뇌는 **draw &lt; target** 만 사용.  
2. 실제 당첨으로 채점 후 DB INSERT.  
3. `update_trust(target)` → `seq_brain.update_model(target)` → `struct_brain.update_models(target + 1)` — **target(및 그 이전)까지 반영하여 이후 회차용**.
