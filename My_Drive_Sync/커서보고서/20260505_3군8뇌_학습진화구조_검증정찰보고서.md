## Q2. 예측이 회차 진행에 따라 변화하는가?

**변화한다 — 메커니즘은 다음의 복합.**

1. **`training` 내용 변화** (`get_v12_training_draws`) — draw 100과 draw 1000에서 **포함 회차·개수·분포**가 다름.  
2. **결정적/준결정적 RNG** — `predict_run.py` 51–66행은 **최근 10회차 해시 시드**; `predict_offset.py` 26–42행 동일; `predict_contrarian.py` 241–248행 **tail 해시**. 같은 논리라도 **입력 tail이 바뀌면 시드·샘플이 바뀜**.  
3. **`random.choices` 등** (`predict_lstm.py` army3 30행, `predict_statistical.py` 샘플링 구간 등) — 실행마다 확률적이나 **PMF는 입력 의존**.  
4. **Hedge 가중치** — 매 회차 예측 저장 후 `update_v12_weights(target_draw_no)` (`v12_engine.py` 358–359행)로 **hyena/snake 합의에 쓰이는 스칼라** 변동 (`_v12_merged_hyena_weights`, `v12_snake` `_load_snake_weights`).  
5. **hyena** — 합의 입력인 **다른 뇌 출력**이 매 회차 바뀌므로 **동일 로직이라도 결과 변화**.  
6. **fusion** — `_v12_load_brain_weights_from_db()` (`v12_fusion_v5.py` 75–94행)로 **DB 가중치** 반영 + `target_draw_no`의 cluster (`96–106행`), 1군 fusion 회피 (`269–270행`).  
7. **LSTM** — `get_lstm_prob_vector`가 데이터 길이·재학습 간격에 따라 **PMF 자체가 변함** (`predict_lstm.py` `RETRAIN_INTERVAL = 50` 39행, `_ensure_model_ready` 196–242행).

**동일 로직 반복**: 각 뇌의 **알고리즘 분기**는 고정 코드이나, **입력·가중치·난수 시드** 때문에 draw 100 vs 1000은 **동일 출력이 보장되지 않음**.  
**캐시**: 이미 40행 이상 저장 시 **`cached` 조기 반환** (`v12_engine.py` 281–287행) — 재실행 시 **예측 물 자체는 안 바뀜**.

## Q3. Hedge 외 뇌 내부 자체 학습 메커니즘?

- **Hedge** (`v12_models.py` `update_v12_weights`): 최근 `last_n` 예측 실적으로 **`exp(η×signal)`·클리핑**으로 **테이블 가중치만** 갱신 — **뇌 내부 신경망 가중치 아님**.

**뇌 내부 “학습”이 있는 경우 (v12_lstm만)** — `app/lotto/predict_lstm.py`:

```135:153:d:\MONEY lol\My_Library\app\lotto\predict_lstm.py
    def _train_on_device(
        model: "LottoLSTM", draws: list[dict], device: Any
    ) -> None:
        ...
        opt = torch.optim.Adam(model.parameters(), lr=1e-3)
        crit = nn.BCELoss()
        model.train()
        for _ in range(EPOCHS):
            for xb, yb in loader:
                opt.zero_grad()
                out = model(xb)
                loss = crit(out, yb)
                loss.backward()

```

- **`torch` 미설치 시**: `get_lstm_prob_vector`는 **균등 PMF** (`predict_lstm.py` 260–263행 `_TORCH_OK` 분기) — **내부 학습 없음**.

**그 외 뇌**: 파라미터 업데이트 없음 — **고정 수식·룰·표본 통계**로 PMF/후보 생성.

**함의**: Hedge만 바뀌면 **hyena/snake가 어떤 뇌 출력을 더 믿을지**는 바뀌지만, **stat/run/offset/contrarian의 생성 규칙 자체**는 바뀌지 않는다(입력 `training` 동일 가정 시). **lstm**만 내부 PMF가 재학습으로 변할 수 있음.

## Q4. 하위 4뇌 avg≈0.78 — 랜덤보다 나은 근거가 코드에 있는가?

- **수학적 기대**: 무작위로 45개 중 6개를 고를 때 **당첨 6개와의 평균 일치 개수** 기대값은 **6×(6/45)=0.8**. 백테 **0.78**은 **이 값과 같은 차수**이다.

- **코드가 하는 일**: “랜덤 초과”를 **명시적으로 주장하지 않음**. 대신:
  - **stat**: 빈도·감쇠·pair 보너스 등 **편향 분포** (`predict_statistical.py` 11–67행).  
  - **run/offset**: 상위 15개 풀에서 **6개 샘플** (`predict_run.py` 114–119행 `pool15`, `rng.choice`). 완전 균등 무작위는 아님.  
  - **contrarian**: D그룹·gap·**1군 Jaccard 회피** 등 **규칙 필터** (`predict_contrarian.py` 249–266행).

**결론**: 코드에는 **“기대 적중 > 무작위”를 검증·보증하는 로직은 없음**. 구현은 **휴리스틱 편향**이며, **관측 avg가 기대 0.8 근처인 것만으로는 코드 상 우월성을 단정할 수 없음**.
