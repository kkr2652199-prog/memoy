        meta_by_tag: dict[str, tuple[float, int, int]] = {}
        for tag, n in n_by.items():

## Q3. 주기적 정규화 설계안 (`w_i' = w_i / Σw * TARGET_SUM`)

### 이론

- **양의 가중치에 대한 동일 비율 스케일**은 **상대 비율**을 보존. 합의가 **가중치 비율만** 쓰면(예: 정규화된 확률·비례 투표), **정규화 직후 합의 결과는 동일**.  
- 문헌상 **Multiplicative Weights / Hedge**는 보통 **손실에 따른 곱셈 갱신**인데, 본 코드는 **시드 기준 `seed·exp(η·signal)` 재계산**에 가깝다. “정규화가 MWU와 수학적으로 동일 계열인가”는 **느슨하게만** 호환(둘 다 가중치 벡터를 시간에 따라 바꿈). **엄밀한 Hedge 복제는 아님**.

### 주기 N

- **너무 짧음**: 클리핑·신호에 의한 단기 격차가 자주 지워져 **학습·차별 신호 약화** 가능.  
- **너무 김**: **100 고착 구간이 길게 반복**.  
- **정찰 권고**: 윈도 **`last_n=50`** 과 맞추 **`N=50`** 을 1차 후보로 두면 운영 의미가 통일됨(동일 시간 스케일).

## 권고안 1개 (확정) + 패치 설계 요약

### 권고: **B안 — 주기적 정규화**를 1순위

- **이유**: `W_MAX`만 올리는 **A**는 재고착·스케일 폭주 리스크가 남고, **D**는 현재 **시드 기반 exp** 구조와 어긋나기 쉬움. **B**는 **비율 보존**으로 합의 해석을 유지하면서 **100에 붙은 표현력을 주기적으로 되살릴** 수 있음.  
- **초기 파라미터**: `NORMALIZE_PERIOD = 50` (`last_n`과 정렬), `TARGET_SUM`은 형이 고정할 값(시드합 vs 운영 타깃)으로 **한 번 결정** 후 백테로 검증.

### 패치 설계 요약(차기 구현용)

1. `target_draw_no % NORMALIZE_PERIOD == 0` (또는 `>0` 조건 추가)일 때만, **갱신 직전 또는 직후** 7뇌 `current_weight`(또는 이번 스텝 `wa`)에 대해 `w_i *= TARGET_SUM / Σw`.  
2. **로그**: 기존 7행 INSERT는 유지 + (가능하면) 정규화 발생 회차에 **부가 플래그** 또는 **보조 테이블 1행**.  
3. **백테**: 동일 초기화 후 `run_v12_chunk_backtest`로 **avg·pvariance·hyena 고착 구간 길이** 비교.  
4. **A안**은 **B 적용 후에도** 상한이 자주 닿으면 **2차**로 `W_MAX` 소폭 상향 검토.
