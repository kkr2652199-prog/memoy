## STEP 1 — `struct_brain` 수정 요약

### 변경 전 (요지)

- `initial_train`: `target_draw < 101`이어도 `_training_matrix` 호출 → **표본 0이면 ValueError**.
- `predict`: 모델 없을 때 `initial_train` 실패 시 **순수 난수 6개 조합**만 반환 (구조 타깃 미사용).

### 변경 후 (요지)

- `initial_train`: `int(target_draw) < MIN_DRAW_TRAIN`이면 **로그(옵션) 후 `return {}`**.
- `_center_y_hat_uniform()` + **`_pick_sets_from_y_hat`**: 기존 `_generate_candidates`·Jaccard·패딩 로직과 동일하게 **구조 소프트매치 기반** 5세트 생성.
- `predict`: 디스크에 모델이 없으면 위 **구조 폴백** 경로 사용.

## STEP 2 — `evolution_brain` 로직 분석 (코드 미변경)

### `update_trust(completed_draw_no, db_path)`

- 다음 회차 예측에 쓰일 **`tw = get_dynamic_weights(completed_draw_no + 1, db_path)`** 를 구한 뒤, `CHIEF_TAGS` 각각에 대해:
  - 해당 회차 **적중 평균** `mc` ← `lotto_fullbacktest_army4` 우선, 없으면 `lotto_predictions_army4` (`matched_count >= 0` 행).
  - **`trust_score`** 컬럼에 위 **`tw.get(tag, 0.25)`** 저장 (컨텍스트 MA5·`matched_count` 동시 기록).

### `get_dynamic_weights(draw_no, db_path)`

- `LOOKBACK=20` 구간에서 과거 회차 `tgt = draw_no - delta`마다 `_avg_matched_for_draw`로 이력 수집.
- 태그별 **`len(pairs) < 3`** 이면 **`trust_raw = 0.25`** (데이터 부족 분기).
- **최소 이력(차등 계산에 들어가려면): 태그당 유효한 과거 평균 적중 레코드 ≥ 3개** (동일 `draw_no`에 대해 풀백·예측 중 하나라도 있어야 함).
- 이후 [0.1, 0.5] 클립 및 재정규화 → dict **키는 `CHIEF_TAGS`만** (`v13_cdm` / `v13_cond_prob` 없음).

### 20회 `update_trust(100..119)` 테스트

- **직전 DB**에 해당 구간 chief 적중 이력이 없거나 대칭이면 **`{'v13_seq': 0.5, 'v13_struct': 0.5}` 유지**.
- 출력: `균등(0.5/0.5)이 아닌지` → **False**
- 테스트 종료 시 **`lotto_evolution_trust_army4` DELETE** (지시대로).
