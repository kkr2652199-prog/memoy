\## 5. 인터넷 정찰 핵심 요약



\### 1티어 다양성 생성 7기법

1\. Different Algorithms — Kaggle 표준

2\. Feature Subset — 부분집합 학습

3\. \*\*Negative Correlation Learning (NCL)\*\* ⭐ — loss 페널티로 다름 강제

4\. \*\*Mixture of Experts (MoE)\*\* ⭐ — 입력 공간 분할 전문가

5\. Random Seed Ensemble — 100시드 평균

6\. Stacking — 다층 앙상블

7\. Pseudo-labeling — ❌ 컨닝 위험으로 우리는 거부



\### 컨닝 방지 정통 (López de Prado)

\- Walk-Forward Validation (시간 순서 유지)

\- Purged Cross-Validation (embargo 갭)

\- No Random Shuffling

\- 우리 현재: target\_draw\_no < N 차단 ✅, embargo 미적용 (검토)



\## 6. 3군 학습 전략 3단 확정



\### Phase 2-A: NCL 페널티 강화 (1순위, 즉시)

\- fusion 학습 loss에 "1군+2군과 다름" 페널티 명시 추가

\- 현재 V12-D 약점 = bypass\_attempts > 30 강제 채움 → NCL로 사전 회피

\- 학습 자체가 다양성 만들어 bypass 불필요

\- 위험 낮음, 효과 높음



\### Phase 2-B: MoE 데이터 분할 (2순위)

\- 학습 데이터 = 1군+2군 미당첨 교집합 986회로 한정

\- 진짜 "틈새 사냥꾼" 포지션 확립

\- 데이터 충분성 미니 백테로 검증 필요



\### Phase 2-C: Algorithm Diversity (3순위, 선택)

\- v12\_lstm → GRU/Transformer

\- v12\_combo → CatBoost

\- v12\_hyena → Extra Trees 5005조합

\- 효과 최대, 구현 복잡, STEP3 이후 진행



\### 거부: Pseudo-labeling

\- 우리 룰(예측→확인→학습)과 충돌

\- 미래 데이터 누수 위험
