안녕하세요. 코딩 못하는 문과 개발자
한글입니다.
클로드 코드 쓰고 계신 분들 한 달에
비용이 얼마나 나오시나요? 엔스로픽
공식 문서에 따르면 개발자 평균이
하루 6달러입니다.
한 달이면 180달러가 넘는
금액입니다. 적지 않습니다. 저도
처음에는 비용을 전혀 신경 쓰지 않고
썼다가 청구서를 보고 깜짝
놀랐습니다. 그런데 오늘 알려 드리는
방법 10가지를 적용하고 나서 비용을
상당히 줄일 수 있었습니다. 어렵지
않습니다. 지금 바로 따라할 수 있는
것들입니다. 하나씩 차근차근 알려
드리겠습니다.
먼저 왜 토큰이 곧 돈인지 간단히
설명드리겠습니다.
클로드 코드를 쓸 때 비용이 발생하는
구조를 이해하면 어디서 절약할 수
있는지가 보입니다. 크게 3 가지
종류의 토큰의 비용이 발생합니다.
첫째, 여러분이 질문을 보낼 때
쓰이는 입력 토큰입니다. 둘째,
클로드가 응답할 때 쓰이는 출력
토큰입니다. 셋째, 클로드가
내부적으로 깊이 생각하는 과정에서
쓰이는 사고 토큰입니다. 특히이 사고
토큰은 출력 토큰과 동일한 가격으로
과금됩니다. 여기서 중요한 점이
있습니다. 대화가 길어질수록 이전에
나눈 모든 대화 내용이 매번 입력
토큰으로 다시 포함됩니다. 처음에는
적은 비용이지만 대화가 쌓이면
눈덩이처럼 불어나는 구조입니다.
영수중에 비유하면 대화가 길어질수록
영수증이 끝없이 길어지는 것과
같습니다. 이러한 구조를 이해하면 왜
토큰 절약이 중요한지 자연스럽게 알
수 있습니다.
그럼 실제로 비용이 얼마나 나올까요?
랜스로픽 공식 문서 기준으로
말씀드리겠습니다. 하루 평균
6달러입니다. 그리고 전체 사용자의
90%는 하루 12달러 이하를
지출하고 있습니다. 손해 모델을 주로
사용하는 경우 월 100달러에서
200달러 수준이라고 합니다. 한
달에 10만 원에서 20만 원 정도
되는 금액입니다. 결코 적지
않습니다. 하지만 걱정하지 마십시오.
지금부터이 비용을 줄이는 10가지
방법을 하나씩 알려 드리겠습니다.
첫 번째 방법 클드 이그어
설정입니다. 이것이 가장 효과가 큰
단일 방법입니다.
클로드 이그노어 파일을 프로젝트 루트
폴더에 만들면 클로드가 해당 파일이나
폴더를 컨텍스트에서 완전히
제외합니다. 문법은 기시근노와
동일합니다. 이미 익숙하신 분들이
많을 것입니다.
화면에 보이는 것처럼 노드 모듈스
단넥스트 필드 디스트 같은 폴더와 락
파일을 추가하면 됩니다. 대형
프로젝트에서는 이것 하나만으로도
컨텍스트가 수십MW 단위로 줄어들 수
있습니다. 설정하데 30초면
충분합니다.
아직 안 만드셨다면이 영상 보시면서
지금 바로 만들어 보시기 바랍니다.
두 번째 방법은 슬래시 클리어와
슬래시 콤팩트 활용입니다.
작업이 바뀔 때마다 슬래시 클리어를
입력하면 대화 기록이 완전히
초기화됩니다.
이전 작업의 컨텍스트가 다음 작업에
불필요하게 딸려 들어가는 것을
막아줍니다.
비화자면 슬래시 클리어는 세금 환급
신청과 비슷합니다. 안 하면 그냥
돈을 버리는 것과 같습니다.
그리고 대화가 길어졌지만 아직 같은
작업을 하고 있다면 슬래시 컴팩트를
입력하면 됩니다. 이전 대화 내용을
요약 압축해서 컨텍스트 크기를 줄여
줍니다.이 두 가지를 습관처럼 쓰는
것이 핵심입니다. 특히 슬래시
클리어는 새로운 작업을 시작할 때
습관적으로 먼저 입력하시는 것을
추천드립니다.
세 번째 방법 작업에 맞는 모델을
고르는 것입니다. 클로드 코드에서
슬래시 모델 명령어를 입력하면 세션
중에 언제든 모델을 바꿀 수
있습니다. 비유하자면 식료품 사러 갈
때 스포츠카를 몰고 갈 필요가 없는
것과 같습니다. 간단한 작업에 비싼
모델을 쓸 이유가 전혀 없습니다.이
한 가지 습관만 들여도 비용이 눈에
띄게 달라집니다. 다음 화면에서
모델별 추천 용도를 구체적으로 보여
드리겠습니다.
구체적으로 보겠습니다. 단순한 조회
포매팅 작업, 이름 변경 같은 반복
작업은 하이크로 충분합니다. 비용이
가장 낮습니다. 테스트 작성이나
일반적인 코딩 작업은 소네이 비용
대비 성능이 가장 좋습니다. 대부분의
일상적인 작업은 소네스로 처리하시면
됩니다. 오퍼스는 복잡한 아키텍처
설계나 여러 파일에 걸친 대규모
리팩터링처럼 정말 어려운 작업에만
쓰면 됩니다. 이렇게 작업 난이도에
따라 모델을 전환해 주는 것만으로도
비용 차이가 상당합니다.네
번째 방법은 정확한 프롬프트를 쓰는
것입니다. 화면 왼쪽이 나쁜
얘입니다. 이거 고쳐줘. 에러는
데처럼 막연하게 쓰면 클로드가
프로젝트 전체를 뒤져야 합니다.
파일을 하나하나 읽고 분석하느라
그만큼 토큰을 낭비하게 됩니다.
반면에 오른쪽처럼 파일 경로와 함수
이름을 정확히 지정하면 클로드가 해당
부분만 바로 확인합니다. 같은 결과를
얻으면서도 토큰 소비량은 크게 차이가
납니다. 프롬프트를 구체적으로 쓰는
습관만 들여도 불필요한 토큰 소비를
상당히 막을 수 있습니다.
다섯 번째 방법 플랜 모디입니다.
시프트 탭을 누르면 플랜 모드가
활성화됩니다.이 모드에서는 클로드가
계획만 수립하고 실제 코드를 수정하지
않습니다. 계획을 먼저 확인하고
방향이 맞으면 그때 실행시키면
됩니다. 만약 계획이 잘못된
방향이라면 스케이프를 눌러서 즉시
중단할 수 있습니다. 실행 전에
중단하면 그만큼 토큰을 아끼는
것입니다. 설계도 없이 집을 짓기
시작하면 비용이 훨씬 많이 드는 것과
같은 원리입니다. 플랫 모드로 먼저
방향을 잡고 시작하면 헛된 토큰
소비를 크게 줄일 수 있습니다.
여기까지 5 가지 방법을 알려
드렸습니다. 클로드 이그노 슬래시
클리어와 슬래시 콤팩트 모델 전환
정확한 프롬프트 그리고 플랫
모드입니다.이
5가지만 적용해도 비용이 눈에 띄게
줄어듭니다. 초보자분들은 여기까지만
따라하셔도 충분합니다. 그런데 다음
오지는 더 강력하고 효과적인
방법들입니다.
계속 보시겠습니다.
여섯 번째 방법. MCP 대신 CLI
도구를 쓰는 것입니다. MCP 서버를
활성화하면 스키마 정보가 매 요청마다
컨텍스트에 포함됩니다. 아무 작업을
안 해도 고정 토큰 비용이 계속
발생하는 구조입니다. 예를 들어
기터브 작업을 할 때 기터브 MCP
대신 GHCLI를 쓰면 동일한 작업을
훨씬 적은 토큰으로 처리할 수
있습니다. 화면에 보이는 것처럼
GHP 리스트나 GH 이슈 크리에이트
같은 명령어를 클로드가 직접 실행하면
됩니다. 일곱 번째 방법은
서브에이전트 활용입니다. 클로드
코드의 서브에이전트는 별도의 컨텍스트
창에서 실행됩니다. 메인 에이전트의
컨텍스트로 소비하지 않는다는
뜻입니다. 큰 작업을 서브에이전트에게
위임하면 메인 컨텍스트가 깨끗하게
유지됩니다. 장기적으로 토큰 자
효과가 매우 큽니다.
여덟 번째와 아홉 번째 방법을 함께
알려 드리겠습니다. 먼저 클로드 MD
파일 최소화입니다. 클로드 MD는
매세션이 시작될 때마다 자동으로
컨텍스트에 로드됩니다.이 파일이
비대하면 매번 불필요한 토큰을
소비하게 됩니다. 커뮤니티에서는
500 토큰 이하로 유지하는 것을
권장하고 있습니다. 꼭 필요한 규칙만
간결하게 적으시기 바랍니다. 다음으로
아홉 번째 방법 익스탠디드 싱킹
조절입니다.
사고 토큰도 출력 토큰과 동일한
가격으로 과금됩니다. 단순한 작업에
깊은 사고가 필요하지 않습니다.
슬래시에트 명령으로 사고 깊이를
로우, 미디엄, 하이 중에서 선택할
수 있습니다. 또한 매스싱킹 토큰스
환경 변수를 설정하면 사고 토큰의
최댓값을 직접 제한할 수도 있습니다.
화면에 보이는 것처럼 8, 정도로
설정하면 적절합니다.
마지막 열 번째 방법은 보너스입니다.
올라마를 이용한 로컬 모델
연결입니다. 단계는 간단합니다. 먼저
올라마를 설치하고 원하는 모델을
다운로드 합니다. 그다음 환경 변수
엔스로픽 베이스 URL을 로컬로스트로
설정하면 클로드 코드 인터페이스에서
로컬 모델을 바로 사용할 수
있습니다. 화면에 보이는 명령어 한
줄이면 설정이 끝납니다. 비용은
0원입니다. 전기세만 나옵니다. 다만
솔직히 말씀드리겠습니다. 품질은
클로드에 비해 확실히 떨어집니다.
복잡한 작업에는 적합하지 않습니다.
하지만 간단한 파일 정리, 포매팅,
이름 변경 같은 단순 작업에는 충분히
활용할 수 있습니다. 무료라는 장점이
확실하기 때문에 한번 시도해 보시는
것을 추천드립니다.
오늘 알려 드린 열 가지 방법
정리하겠습니다. 첫째, 클로드어로
불필요한 파일 제외. 둘째,
슬래시클리어와 슬래시 컴팩트로
컨텍스트 관리. 셋째, 작업별 모델
전환. 넷째, 정확한 프롬프트 작성.
다섯째, 플랜 모드 활용
우선 사용 일곱째, 서브에트 위임.
여덟째 클로드 MD 최소화. 아홉째
익스텐디드 싱킹 조절. 그리고 열제
올라마 로컬 모델입니다. 하나씩
차근차근 적용해 보시기 바랍니다.이
중에서 지금 바로 적용할 수 있는
것이 몇 개인지 댓글로 알려 주시면
감사하겠습니다. 영상이 도움이
되셨다면 구독과 좋아요 부탁드립니다.
시청해 주셔서 감사합니다.
